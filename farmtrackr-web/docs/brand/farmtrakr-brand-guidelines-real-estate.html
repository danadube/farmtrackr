Our voice is professional yet approachable‚Äîlike a knowledgeable colleague who's always willing to help. We speak plainly, avoid jargon, and focus on how we help agents succeed.

Writing Guidelines

‚úì
Do
  • Use active voice ("Track your clients")
  • Start with verbs for actions
  • Use contractions (we're, you'll, they've)
  • Address users directly ("your listings")
  • Keep sentences short and scannable
  • Focus on benefits, not features

Don't

  • Use industry jargon or acronyms
  • Write in third person
  • Use unnecessarily complex words
  • Write long paragraphs
  • Focus only on what the product does

Design Principles

Our design philosophy balances sophistication with simplicity. Whether using the Apple Aesthetic or Orchard Grove approach, these principles guide all design decisions.

1. Clarity Above All

Every interface element should have a clear purpose. If a feature doesn't help agents manage their properties more effectively, it shouldn't be there. Remove decoration; add value.

2. Respect the User's Time

Real estate agents are busy. Design for efficiency. Common tasks should take minimal clicks. Critical information should be visible at a glance. Loading states should reassure, not frustrate.

3. Consistent, Not Boring

Use patterns and conventions consistently, but don't sacrifice visual interest. Our color-coding system provides functional variety within a consistent framework. Patterns create familiarity; color creates engagement.

4. Mobile-First, Always

Agents work on the go‚Äîat showings, open houses, and client meetings. Design for mobile first, then scale up. Touch targets must be large enough. Text must be readable without zooming. Actions must work with one hand.

5. Accessible by Default

Accessibility isn't an add-on. Maintain 4.5:1 contrast ratios. Support keyboard navigation. Provide alt text. Design for screen readers. Everyone deserves great tools.

6. Delight in the Details

Smooth animations. Subtle hover effects. Thoughtful micro-interactions. These details don't just look good‚Äîthey communicate that we care about quality and that agents can trust our platform.

Design System Specifications

Element Specification Notes
Grid System 8-point base unit All spacing in multiples of 8px (8, 16, 24, 32...)
Border Radius Small: 8px, Medium: 12px, Large: 16px Consistent rounded corners throughout
Shadows Subtle, layered (Apple) or standard (Orchard) Never harsh; create depth not distraction
Animation Duration 200-300ms for interactions Fast enough to feel responsive, slow enough to see
Animation Easing cubic-bezier(0.4, 0, 0.2, 1) Apple-style smooth deceleration
Touch Targets Minimum 44x44px Following iOS Human Interface Guidelines
Max Content Width 1400px Optimal reading and scanning width
